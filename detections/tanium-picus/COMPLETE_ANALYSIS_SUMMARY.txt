ACTUAL BEHAVIORAL PATTERNS IDENTIFIED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

From analysis of Tanium alerts and Picus attack specifications:

1. REGISTRY PERSISTENCE
   Process: reg.exe, powershell.exe, cmd.exe
   Behavior: Create/Modify HKCU\Software\Microsoft\Windows\CurrentVersion\Run*
   Instances: 35 undetected actions
   Observable: Registry operation + process + non-System context
   MITRE: T1547, T1037, T1547.001

2. COM OBJECT EXPLOITATION (LARGEST GAP)
   Process: powershell.exe, wscript.exe, cscript.exe
   Behavior: New-Object/CreateObject for COM classes (MMC, Word, Excel, WinRM)
   Instances: 91 undetected actions
   Observable: PowerShell cmdlet + COM class name + SYSTEM-level access from user
   MITRE: T1021.003, T1559.002
   Example Attack: "Execute Invoke-COM-ShellBrowserWindow Function from COM-Utility.ps1"
   Status: NOT DETECTED by Tanium

3. SCHEDULED TASK CREATION
   Process: powershell.exe (NO schtasks.exe parent), schtasks.exe
   Behavior: Register-ScheduledTask cmdlet or schtasks.exe /create
   Instances: 18 undetected actions
   Observable: PowerShell Register-ScheduledTask OR schtasks.exe with /create parameter
   MITRE: T1053.005
   Gap: Only 5x generic "Script Based Scheduled Tasks" detected (23 total attempts)

4. WMI LATERAL MOVEMENT
   Process: powershell.exe, cmd.exe, wmic.exe
   Behavior: Invoke-WmiMethod, Get-WmiObject Win32_Process/Service, wmic.exe
   Instances: 11 undetected actions
   Observable: PowerShell WMI cmdlets or wmic.exe execution
   MITRE: T1047, T1059.001
   Example: "Execute an Arbitrary Command by using the Invoke-WmiMethod" - NOT DETECTED

5. CREDENTIAL DUMPING
   Process: powershell.exe, cmd.exe, certutil.exe, custom tools
   Behavior: Access to HKLM\SAM, HKLM\SECURITY, HKLM\SYSTEM registry hives
   Instances: 8 undetected actions + multiple tools
   Observable: Registry hive access from non-System context
   MITRE: T1003, T1003.002
   Examples:
   - "Dump SAM, SECURITY and SYSTEM Registry Hives via fodhelper.exe" - NOT DETECTED
   - "Dump Credentials by using LaZagne Tool" - Possibly DETECTED as Process Injection

6. SHADOW COPY/BACKUP DELETION
   Process: vssadmin.exe, powershell.exe (WMI), cmd.exe
   Behavior: vssadmin delete shadows, WMI volume operations
   Instances: 7 undetected actions
   Observable: vssadmin.exe or WMI methods targeting volume shadow copies
   MITRE: T1490, T1047
   Alert Status: 15x "Volume Shadow Copy Deletion" BUT may not cover all methods

7. LOG CLEARING (ADVANCED METHODS)
   Process: wevtutil.exe, powershell.exe, cmd.exe, fsutil.exe
   Behavior: wevtutil.exe, PowerShell Clear-EventLog, fsutil usn deletejournal
   Instances: 10+ undetected actions
   Observable: Event log clearing OR fsutil operations OR ETW/trace disabling
   MITRE: T1070, T1070.004
   Alert Status: 14x "Windows Event Logs Cleared" BUT misses advanced methods

1. DETECTION IS NOT BEHAVIOR-AWARE
   Current Tanium signals detect PROCESS EXECUTION but not BEHAVIORAL CONTEXT
   Example: "Suspicious PowerShell" fires 22x but misses:
   - Register-ScheduledTask execution (persistence)
   - New-Object -ComObject exploitation (DCOM RCE)
   - Invoke-WmiMethod execution (lateral movement)

   Fix: Create BEHAVIORAL rules that correlate process + specific cmdlet patterns

2. REGISTRY MODIFICATIONS ARE UNDER-DETECTED
   reg.exe executed 28x but only 8 relevant detections (28.6% coverage)
   Missing: Run, RunOnce, Service, Office persistence patterns

   Fix: Implement 5+ registry-specific persistence detection rules

3. COM OBJECT EXPLOITATION IS CRITICAL GAP
   91 undetected DCOM/COM object exploitation actions
   This is the LARGEST single detection gap
   Affects: Lateral movement, RCE, persistence

   Fix: Create comprehensive COM object detection rule with risk classification

4. NATIVE TOOLS HIDE ATTACKS
   PowerShell, cmd.exe, reg.exe are used in 90%+ of attacks
   These are legitimate tools, making detection hard
   Solution is NOT to block them, but to detect BEHAVIORS they enable

   Fix: Behavior-centric rules (cmdlet patterns, registry paths, API calls)

5. TEMPORAL CLUSTERING
   Attacks were executed in concentrated waves (20-23 UTC)
   Suggests batched reconnaissance followed by exploitation
   Opportunity: Correlate attack wave patterns for automated response

   Fix: Implement timeline-aware detection correlation
